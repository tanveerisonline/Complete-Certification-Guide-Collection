# CompTIA Network+ Certification Guide (2025-2026)

## Overview
CompTIA Network+ is an intermediate-level certification validating networking knowledge and skills. It covers network architecture, installation, configuration, maintenance, and troubleshooting of network hardware and software.

**Certification Code:** N10-008

---

## Duration & Study Timeline
- **Recommended Study Duration:** 3-4 months
- **Study Hours Required:** 200-250 hours
- **Exam Preparation Time:** 6-8 weeks after completing course material

---

## Exam Details

### Exam Duration
- **Total Duration:** 90 minutes
- **Total Questions:** 80 questions
- **Question Format:** Multiple choice, drag-and-drop, simulation, hotspot

### Exam Structure
- **Passing Score:** 720/900
- **Score Range:** 100-900 points
- **Difficulty Level:** Intermediate

### Exam Fee (2025-2026)
- **Exam Fee:** $238 USD
- **Retake Fee:** $238 USD
- **Bundle with Security+:** Available from authorized providers
- **Validity:** Exam must be taken within 2 years of registration

## Complete Syllabus & Topics

### Domain 1: Networking Fundamentals (23%)

#### Network Concepts
- **OSI Model (7 Layers)**
  - Layer 1: Physical (cables, connectors, hubs)
  - Layer 2: Data Link (MAC addresses, switches)
  - Layer 3: Network (IP routing, routers)
  - Layer 4: Transport (TCP, UDP)
  - Layer 5: Session (session management)
  - Layer 6: Presentation (encryption, compression)
  - Layer 7: Application (HTTP, SMTP, FTP)
- **TCP/IP Model**
  - Link layer
  - Internet layer
  - Transport layer
  - Application layer
- **Network Topologies**
  - Bus topology
  - Ring topology
  - Mesh topology (full, partial)
  - Star topology
  - Hybrid topologies

#### IP Addressing & Subnetting
- **IPv4 Addressing**
  - Classful addressing (Class A, B, C, D, E)
  - Classless Inter-Domain Routing (CIDR)
  - Subnetting and subnet masks
  - Network address translation (NAT)
  - DHCP and static addressing
- **IPv6 Addressing**
  - IPv6 address format
  - IPv6 prefixes
  - IPv6 features and benefits
  - IPv6 transition mechanisms
- **Ports and Protocols**
  - Well-known ports (1-1023)
  - Registered ports (1024-49151)
  - Dynamic/private ports (49152-65535)

#### Networking Equipment
- **Switches**
  - Switch functions and operations
  - VLAN basics
  - Spanning Tree Protocol (STP)
  - Switching types (cut-through, store-and-forward)
- **Routers**
  - Router functions
  - Static and dynamic routing basics
  - Default gateways
  - Routing tables
- **Network Appliances**
  - Firewalls
  - Proxy servers
  - Load balancers
  - VPN appliances

### Domain 2: Network Implementations (20%)

#### Ethernet Networking
- **Ethernet Standards**
  - 100BASE-T (Fast Ethernet)
  - 1000BASE-T (Gigabit Ethernet)
  - 10GBASE-T (10 Gigabit)
  - PoE (Power over Ethernet)
- **Cabling**
  - Cat5e, Cat6, Cat6a, Cat7, Cat8
  - Fiber optic cables
  - Cable ratings and certifications
  - Cable management best practices

#### Wireless Networking
- **WiFi Standards (802.11)**
  - 802.11a/b/g (legacy)
  - 802.11n (WiFi 4)
  - 802.11ac (WiFi 5)
  - 802.11ax (WiFi 6)
  - 802.11be (WiFi 7 - new for 2025)
- **Wireless Security**
  - WPA2 and WPA3 protocols
  - Encryption standards
  - Authentication methods
  - Rogue AP detection
- **Wireless Site Surveys**
  - Coverage and capacity planning
  - Interference detection
  - Signal strength measurement
  - Heat mapping

#### Wide Area Networks (WAN)
- **WAN Technologies**
  - Metro Ethernet
  - MPLS
  - VPN (Virtual Private Networks)
  - SD-WAN (Software-Defined WAN)
  - DIA (Direct Internet Access)
- **Internet Connectivity**
  - Cable broadband
  - DSL
  - Fiber
  - Wireless WAN (5G, LTE)
  - Satellite internet

### Domain 3: Network Operations (16%)

#### Network Services
- **DHCP (Dynamic Host Configuration Protocol)**
  - DHCP scopes and leases
  - DHCP servers and relays
  - DHCP options
  - IPv6 DHCP (DHCPv6)
- **DNS (Domain Name System)**
  - DNS hierarchy and zones
  - DNS records (A, AAAA, CNAME, MX)
  - DNS servers and clients
  - DNSSEC
  - DNS troubleshooting
- **Network Time Protocol (NTP)**
  - Time synchronization
  - NTP servers and clients
  - Time zones and daylight saving
- **SNMP (Simple Network Management Protocol)**
  - SNMP agents and managers
  - SNMP versions (v1, v2, v3)
  - MIBs (Management Information Base)
  - SNMP traps and alerts
- **Syslog**
  - Log aggregation
  - Log levels and facilities
  - Syslog servers
  - Log analysis

#### Network Monitoring & Telemetry
- **Monitoring Tools**
  - Ping and traceroute
  - Netstat and ss
  - tcpdump and Wireshark
  - IPAM tools
  - Network analyzers
- **NetFlow & Sflow**
  - Flow data collection
  - Traffic analysis
  - Capacity planning with flow data
- **SNMP Monitoring**
  - Performance metrics
  - Alerts and thresholds
  - MIB exploration

### Domain 4: Network Security (20%)

#### Security Fundamentals
- **Access Control Lists (ACLs)**
  - Inbound and outbound rules
  - Standard vs extended ACLs
  - ACL implementation
  - IPv6 ACLs
- **Firewalls**
  - Stateless firewalls
  - Stateful firewalls
  - Application firewalls (WAF)
  - Next-generation firewalls
  - Firewall zones and demilitarized zones (DMZ)

#### Encryption & VPN
- **Encryption Protocols**
  - IPSec (IKE, AH, ESP)
  - TLS/SSL
  - AES and other algorithms
- **VPN Technologies**
  - Site-to-site VPN
  - Remote access VPN
  - VPN clients and servers
  - Split tunneling
  - VPN monitoring

#### Identity & Authentication
- **AAA (Authentication, Authorization, Accounting)**
  - RADIUS
  - TACACS+
  - LDAP and Active Directory integration
  - 802.1X authentication
  - MFA concepts
- **Certificates & PKI**
  - Certificate authority (CA)
  - Public and private keys
  - Certificate lifecycle
  - CRL and OCSP

#### Network Security Design
- **Segmentation**
  - Network segmentation strategies
  - VLAN security
  - Microsegmentation
  - Zero Trust architecture
- **Threat Mitigation**
  - DDoS protection
  - IDS/IPS (Intrusion Detection/Prevention)
  - Sandboxing
  - Honeypots and honeynets

### Domain 5: Network Troubleshooting & Tools (21%)

#### Troubleshooting Methodology
- **Systematic Approach**
  1. Identify the problem
  2. Establish a theory
  3. Test the theory
  4. Establish action plan
  5. Implement solution
  6. Verify solution
  7. Document findings

#### Troubleshooting Scenarios
- **No Connectivity Issues**
  - Physical layer problems
  - DHCP failures
  - IP addressing conflicts
  - Default gateway issues
- **Intermittent Connectivity**
  - Wireless interference
  - Port flapping
  - Duplex mismatches
  - MTU issues
- **Poor Performance**
  - Bandwidth saturation
  - Latency problems
  - Jitter issues
  - Packet loss
  - Congestion

#### Network Troubleshooting Tools
- **Command-Line Tools**
  - ping and ping6
  - tracert/traceroute
  - nslookup and dig
  - ipconfig/ifconfig
  - netstat and ss
  - arp and route
  - ip commands
- **Protocol Analyzers**
  - Wireshark
  - tcpdump
  - Packet capture analysis
- **Network Monitoring Tools**
  - Interface monitoring
  - Flow monitoring
  - Syslog analysis
  - SNMP monitoring
- **Connectivity Tools**
  - Cable testers
  - Tone generators and probes
  - Optical power meters (for fiber)
  - Network multimeters

#### Common Network Problems
- **Physical Layer Issues**
  - Bad cables
  - Damaged connectors
  - Port failures
  - Fiber cuts
- **Link Layer Issues**
  - MAC address problems
  - VLAN misconfigurations
  - STP loops
  - Broadcast storms
- **Network Layer Issues**
  - Routing problems
  - IP misconfiguration
  - NAT translation failures
- **Application Layer Issues**
  - Service failures
  - DNS resolution problems
  - Slow application response

## Tips for Success

1. **Hands-On Practice:** Use Packet Tracer and GNS3 extensively
2. **Understand Protocols:** Don't just memorize; understand how protocols work
3. **Practice Subnetting:** Master subnetting calculations
4. **Study in Blocks:** Focus on one domain at a time
5. **Real-World Examples:** Relate concepts to actual network scenarios
6. **Take Multiple Practice Exams:** Aim for 80%+ consistently
7. **Join Study Groups:** Collaborate with peers
8. **Stay Current:** Follow networking trends and updates
9. **Review Weak Areas:** Spend extra time on difficult topics
10. **Get Certified:** Complete the certification to validate knowledge

## Exam Tips & Strategy

- **Time Management:** Allocate ~1 minute per question
- **Read Carefully:** Network+ questions can be tricky
- **Use Elimination:** Remove obviously wrong answers
- **Flag Difficult Questions:** Mark and return to them
- **Simulation Practice:** Get comfortable with interactive questions
- **Don't Second-Guess:** Trust your training and first instinct
- **Manage Stress:** Stay calm during the exam
